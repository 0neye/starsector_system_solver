# Starsector System Ranker v0.1.0

First public release. A colony-planning solver and ranker for
[Starsector](https://fractalsoftworks.com/): point it at your save and it ranks
your discovered systems by colony potential, then computes near time-optimal
build plans to hit an income / stability / defense goal in the fewest game
months. It reads your local game and save files only — it never modifies your
game.

> **Unofficial fan-made tool.** Not affiliated with, endorsed by, or supported
> by Fractal Softworks. You must own a copy of Starsector to use it.

## Highlights

- **Save extraction** — parses a Starsector save into a local SQLite database
  (`extract run`), with save listing, search, and export. All solver modes read
  from this DB.
- **System ranking** (`--rank`) — scores every discovered system best-first
  using a floor-aware, credit-relaxed upper bound (the default `bound` scorer),
  with `quick` and `template` scorers as alternatives. Scope filters limit
  ranking to systems you've actually surveyed.
- **Goal solver** (`--solve`) — finds a near time-optimal sequence of colony
  actions (colonize, build facilities, install items/cores, set free port /
  hazard pay) to reach an income / stability / defense goal, with a Pareto
  frontier and a recommended balanced plan.
- **Maximize mode** (`--maximize income|stability|defense`) — pushes one metric
  within a horizon while holding the others as floors.
- **Interactive TUI** (`tui`) — Saves → Rank → System → Solve → Plan, with
  discovered-only scoping, background rank/solve/extract jobs with cooperative
  cancel, and balance seeded from the loaded save.
- **Native self-installer** — `system_solver install` (wrapped by `install.ps1`
  / `install.sh`) copies the binary to a per-user location, adds it to `PATH`,
  runs an initial extraction, and creates a launcher. No Python or other runtime
  required.

## Economy & solver model

- Commodity export income is pooled at **system scope**: all player producers of
  a commodity share one market pool, so duplicate industries see diminishing
  returns. Income is non-additive across planets, but weighted supply is.
- Two-level search: a plan level (per-planet decisions) over a schedule level (a
  greedy build-ASAP simulation that models shared resource contention across
  planets for credits, story points, and alpha cores).
- `--time-limit <MS>` is a hard wall-clock deadline; climbs poll it at node
  granularity and return their best-so-far plan. Results are deterministic when
  the solve finishes inside the budget; a cutoff is machine-dependent and
  reported.
- Story-point improvements and industry/structure alpha-core installs are
  searched by default; disable with `--no-industry-upgrades`.

## Install

Download the archive for your OS below, unpack it, and run the bundled installer
from inside the unpacked folder:

- **Windows** (`...-windows.zip`): run `.\install.ps1`
- **Linux** (`...-linux.tar.gz`): run `./install.sh`

Then open a new terminal and run `system_solver tui`. See the
[README](../../README.md) for installer flags and the
[CLI reference](../CLI.md) for the full command set.

## Requirements

- A Starsector installation (auto-detected, or set with `--starsector-dir` /
  `STARSECTOR_DIR`).
- Building from source: a Rust toolchain (1.74+). SQLite is statically bundled.

## Platforms

Prebuilt binaries: Windows (`x86_64-pc-windows-msvc`) and Linux
(`x86_64-unknown-linux-musl`).
