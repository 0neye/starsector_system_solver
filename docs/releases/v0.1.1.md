# Starsector System Ranker v0.1.1

Bug-fix release.

## Fixes

- **First-run install no longer fails to build its database.** On a clean
  install, extraction targets a fresh per-user data directory (e.g.
  `%APPDATA%\StarsectorSystemRanker\data\save_data.db`). SQLite does not create
  intermediate directories, so the very first `extract run` — including the one
  the installer performs automatically — failed with
  `error: DB error: unable to open database file`. `Db::open` now creates the
  database's parent directory before opening it. (#1)

## Upgrading

Unpack this release and run the installer again, or run `system_solver install`.
No data migration is needed; existing databases are unaffected.

See [v0.1.0](v0.1.0.md) for the initial feature set.
